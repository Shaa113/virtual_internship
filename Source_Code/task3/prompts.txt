Task 3: Prompt Engineering Challenge (10 Marks) Design prompts that generate: 1. Clean code 2. Optimized code 3. Beginner-friendly explanations 4. Production-level documentation Example Poor Prompt: Write Python code for sorting. Improved Prompt: Write an optimized Python implementation of Merge Sort with comments, edge-case handling, time complexity explanation, and test cases. Students must: ● Write 5 poor prompts ● Convert them into effective prompts ● Show output improvements
Soln. Prompt 1
Write sorting code.
Improved Prompt 1
You are a senior Python developer and you have to implement the Merge Sort algorithm. The context is that the code is intended for second-year engineering students learning Data Structures and Algorithms. The constraints are:
- Use Python.
- Include comments explaining each step.
- Handle edge cases such as empty arrays and single-element arrays.
- Follow clean coding practices.
And output Format should strictly be:
1. Source Code
2. Time Complexity
3. Space Complexity
4. Sample Input and Output
5. Brief Explanation

Prompt 3
Explain Binary Search.
Improved Prompt
Role:
You are a university professor teaching Data Structures, explain Binary Search. The context is :
The audience consists of first-year engineering students with basic programming knowledge.
The Constraints are:
•	Use simple language.
•	Include a real-life analogy.
•	Explain step-by-step.
•	Include pseudocode.
•	Explain time complexity.
Output Format should be:
1.	Definition
2.	Real-Life Example
3.	Working Steps
4.	Pseudocode
5.	Complexity Analysis
Output Improvement
The improved prompt generated a structured explanation with examples and pseudocode, making the concept easier for beginners to understand.
